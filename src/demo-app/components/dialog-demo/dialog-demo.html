Dialog

Dialogs inform users about a specific task, require decisions, or involve multiple tasks.
A dialog is opened by calling the open method with a component to be loaded and an optional config object. The open method will return an instance of MatDialogRef:
Service
mdc-dialog
Dialog Methods Parameters
open(componentOrTemplateRef: any) Opens a modal dialog containing the given component.
Extra configuration options.
close() Closes the dialog.
Config Description
clickOutsideToClose: boolean Close the dialog if user clicks on the backdrop. (Default: true)
escapeToClose: boolean Enable or disable close on escape key (Default: true).
Directives
mdc-dialog Defines the dialog content block.
mdc-dialog-header Defines the header content block.
mdc-dialog-header-title Briefly describes the type of choice being made.
mdc-dialog-body Body content typically consists of text and/or UI control elements.
mdc-dialog-footer Place mdc-dialog-button's inside the footer.
Events Description
accept Broadcast when a user accepts the dialog.
cancel Broadcast when a user cancels the dialog.
Button Directives
button[mdc-dialog-button]
a[mdc-dialog-button]
Properties Description
accept: boolean Closes the dialog and emits an accept() event.
cancel: boolean Closes the dialog and emits an cancel() event.
action: boolean Apply contrasting secondary color to button.
focused: boolean Set initial focus to button after Dialog opened.

Configuring dialog content via entryComponents

Because MdcDialog instantiates components at run-time, the Angular compiler needs extra information to create the necessary ComponentFactory for your dialog content component. For any component loaded into a dialog, you must include your component class in the list of entryComponents in your NgModule definition so that the Angular compiler knows to create the ComponentFactory.

import  from '@angular/forms';
